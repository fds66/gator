# Blog Aggregator

This blog aggregator ‘gator’ is a CLI tool that allows a user to create a list of feeds to follow and aggregates the posts from these feeds on a continuous basis. Users can follow feeds other users have added. The posts can be browsed and the title, link and description are summarised in the terminal allowing the user to follow the link to view the full post.

The data is stored locally in a postgreSQL data base. The migrations were handled in goose and the queries were generated by sqlc.

## Installation Requirements

In order to install it you will need go version 1.26.1 or later and Postgres v15 or later.

A configuration file is required in your home directory ~/.gatorconfig.json which contains the following structure

```markdown
{
  "db_url": "connection_string_goes_here",
  "current_user_name": "username_goes_here"
}
```

The connection string for the database needs to be provided , the current user is changed within the program using the login command. 

```markdown
connection string syntax
protocol://username:password@host:port/database?sslmode=disable
```

To install the program 

```markdown
go install .  // in the gator root directory
```

## Usage

```markdown
gator register <username>             // registers a user name
gator login <username>                // for an already registered user assigns username as current user
gator users                           // lists all the registered users
gator addfeed <feed name> <feed url>  // adds a feed
gator feeds                           // lists all the feeds in the database
gator follow <feed url>               // creates a follow for the current user for the specified feed
gator unfollow <feed url>             // unfollows a feed
gator following                       // lists all the feeds followed by the current user
gator agg <time interval eg 10m, 20s> // aggregates the posts from the followed feeds at the given interval
gator browse <(optional) num posts>   // prints the details of all the posts that have been aggregated for the current user
																				// provides number of posts specified, default to 2

```

The post aggregation runs continuously so to interrupt the aggregation you need to interrupt the process eg with ctrl c.
